## Description

The posts rendering layer (src/pages/posts/[slug].astro) currently has no navigation between posts. When reading a post, there's no way to go to the next or previous post without going back to the listing.

Add prev/next links at the bottom of each post page. Follow the pattern used by src/layouts/Longform.astro for deep-dive sibling navigation (queries collection, sorts, finds current entry, renders prev/next links), but keep it simpler for posts.

Implementation approach:
- In src/pages/posts/[slug].astro, query the posts collection, sort by date descending
- Find the current post's position in the sorted list
- Pass prevPost and nextPost to the layout (or render nav directly after `<RenderedContent />`)
- Render a simple prev/next bar at the bottom: left arrow + previous post title on the left, next post title + right arrow on the right
- Style using existing CSS custom properties (var(--color-text-muted), var(--color-border), etc.)

This is a framework file, so it must work generically for any Loomwork site with posts. No site-specific hardcoding.


## Acceptance Criteria

- [ ] Post pages display prev/next navigation links at the bottom
- [ ] "Previous" links to the older post (earlier date), "Next" links to the newer post (later date)
- [ ] First post (oldest) has no prev link; most recent post has no next link
- [ ] Navigation styling matches existing site patterns using CSS custom properties
- [ ] No em dashes in any copy
- [ ] No site-specific hardcoding; works generically for any Loomwork site with posts
- [ ] npm run build passes
